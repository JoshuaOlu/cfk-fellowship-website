# CFK Fellowship Website Design System

Version: 1.0 (Draft)

## Purpose

The CFK Fellowship website should communicate credibility, optimism, professionalism, and openness. It is not simply a recruitment page—it is the digital home of the Fellowship.

Every design decision should reinforce the Fellowship's mission of developing people who create educational opportunities.

---

# Design Principles

## 1. Content First

Content is the product.

Design should amplify the message, never compete with it.

---

## 2. Fast by Default

The website should load quickly, even on slow mobile connections.

We avoid unnecessary JavaScript, large dependencies, and excessive animations.

---

## 3. Mobile First

Every page is designed for mobile devices before larger screens.

Responsive enhancements are added progressively.

---

## 4. Accessible by Default

Accessibility is a requirement, not an enhancement.

The website should support:

- semantic HTML
- keyboard navigation
- sufficient colour contrast
- descriptive alternative text
- clear focus states

---

## 5. Component Driven

Reusable components are preferred over repeated markup.

Common interface elements belong in `_includes`.

---

## 6. Consistent

Spacing, typography, colours, and layouts should come from the design system.

Avoid one-off styling whenever possible.

---

# Typography

## Headings

Syne

Used for:

- page titles
- section headings
- emphasis

## Body

DM Sans

Used for:

- paragraphs
- buttons
- navigation
- forms

---

# Colour Philosophy

Blue represents trust, growth, and knowledge.

Neutral colours should dominate.

Accent colours should be used intentionally.

---

# Layout Philosophy

Maximum content width:

1200px

Whitespace is part of the design.

Sections should feel spacious rather than crowded.

---

# Performance Goals

Lighthouse targets:

- Performance ≥ 95
- Accessibility = 100
- Best Practices = 100
- SEO = 100

---

# Browser Support

Support modern evergreen browsers.

Progressive enhancement is preferred over polyfills.

---

# Repository Standards

- HTML should be semantic.
- CSS should use variables.
- JavaScript should be minimal.
- Images should be optimized.
- Components should be reusable.
